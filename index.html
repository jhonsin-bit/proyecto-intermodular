Proyecto Intermodular SMR

Guía paso a paso para implementar una infraestructura completa basada en Linux

19 de Marzo de 2026

01. Introducción y Objetivos

Este proyecto tiene como objetivo principal implementar una infraestructura completa y robusta basada en Linux.

Los pilares fundamentales en los que nos basamos son:

  • Gestión centralizada y segura de usuarios.
  • Prácticas avanzadas de seguridad.
  • Digitalización integral de procesos.

02. Instalación del Sistema Operativo

Para comenzar nuestra infraestructura, preparamos un entorno de servidor base utilizando Ubuntu Server o Debian.

Pasos realizados:

  1. Descarga de la imagen ISO oficial (Ubuntu Server / Debian).
  2. Instalación y configuración en una máquina virtual.
  3. Configuración de particiones de disco de forma óptima para servidores.

03. Gestión de Usuarios y Grupos

Una correcta administración de usuarios es vital para la seguridad. Aquí se muestra cómo crear y administrar accesos limitando los privilegios.

Comandos principales utilizados:

sudo adduser [nombre_usuario]
sudo groupadd [nombre_grupo]
sudo usermod -aG [grupo] [usuario]

Este proceso garantiza que los nuevos usuarios sean creados con los permisos justos, asignando su directorio de inicio y forzando políticas de creación responsable.

04. Configuración de Cuotas de Disco

Para evitar que un usuario consuma todo el almacenamiento del servidor, implementamos un sistema de cuotas (Quotas).

Pasos clave:

  1. Edición del archivo de configuración /etc/fstab para habilitar cuotas en la partición adecuada.
  2. Uso de quotacheck para escanear y generar los archivos base.
  3. Configuración individual mediante edquota.

05. Implementación del Sistema de Tickets

Digitalizamos la gestión de incidencias instalando un software de Helpdesk / Gestión de tickets. El software elegido podía ser GLPI, Odoo, u osTicket.

Flujo de trabajo:

  • Despliegue e instalación en el servidor Linux usando Apache/MariaDB.
  • Configuración de la conexión a la base de datos (creación de usuario y DB).
  • Creación de los usuarios administradores y técnicos para la gestión de tickets.

06. Auditoría de Seguridad y Resultados

Tras montar la infraestructura, verificamos su resiliencia utilizando herramientas profesionales de pentesting y escaneo de red.

Se realizó un escaneo con Nmap y WPScan en la red local para verificar puertos abiertos, servicios en ejecución, y vulnerabilidades.


¡Éxito en la Implementación!

El sistema funciona correctamente, cumpliendo todos los objetivos iniciales trazados de seguridad, gestión y digitalización de procesos.
